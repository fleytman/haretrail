# Contribution Log

This skill handles contribution logs, self-review ledgers and visibility artifacts.

Before working:

- Read `../_shared/system-behavior.md` as the shared reusable behavior contract.

- Read `references/workflow.md`.
- Use `work-artifacts/README.md` as the general container model.
- Do not confuse a contribution log with `debrief`, `lessons` or an ordinary task execution log.

Use this skill when the user asks to:

- record a contribution;
- update a self-review log;
- start or maintain a "book of good deeds";
- record invisible / glue work;
- record whom the user helped and who helped the user;
- preserve started-but-not-finished but significant work;
- prepare a packet of artifacts for a self-review, performance review or a conversation about contribution.
