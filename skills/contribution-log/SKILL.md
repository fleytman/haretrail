# Contribution Log

Этот skill обслуживает contribution logs, self-review ledgers и visibility artifacts.

Перед работой:

- Прочитай `../_shared/system-behavior.md` как общий reusable behavior contract.

- Прочитай `references/workflow.md`.
- Используй `work-artifacts/README.md` как общую модель контейнера.
- Не путай contribution log с `debrief`, `lessons` или обычным task execution log.

Используй этот skill, когда пользователь просит:

- записать вклад;
- обновить self-review log;
- завести или поддерживать "книгу добрых дел";
- фиксировать invisible / glue work;
- записать, кому пользователь помог и кто помог пользователю;
- сохранить started-but-not-finished, но значимую работу;
- подготовить пакет артефактов для self-review, performance review или разговора о вкладе.
