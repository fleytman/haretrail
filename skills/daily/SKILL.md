# Daily

Тонкий потребитель движка `work-evidence`. Собирает рабочий daily-стендап за короткий период
и рендерит его в 4 секции, готовые к дейли/синку.

Перед работой:

- Прочитай `../_shared/system-behavior.md` как общий reusable behavior contract.
- Прочитай `references/workflow.md`.
- Сбор evidence делает движок — прочитай `../work-evidence/references/workflow.md`.
- Каденс дейли, источники, язык и фильтр «личное/работа» бери из конфига `~/.haretrail/`.

Используй этот skill, когда пользователь просит:

- «собери дейли» / «что я делал с X по Y» / «пункты к стендапу»;
- свести работу за день/несколько дней в готовые буллеты;
- не потерять мелкую реальную работу (ревью, draft-PR, координацию, мелкие чистки).

Ключевые принципы:

- **Evidence-first через `work-evidence`**; только рабочие задачи (личное фильтруется).
- **Вывод — 4 секции:** 1) прогресс по задачам · 2) активности вне задач (invisible work) ·
  3) инсайты/проблемы/вопросы · 4) блокеры. Внутри §1 группировать по тикету.
- **Явная атрибуция (КТО ЧТО)**: кто действовал, кто кому помогал. Ревью чужих PR → §2.
- **Статус-путь и кто двигал тикет**: кто нажал кнопку ≠ чьё решение (см. work-evidence).
- **Тон отчёта**: стендап могут показать лиду — нейтрально, без сленга. «помог» только при
  подтверждённом результате, иначе «помогал»/«подсказал».
- **Чек уроков/дебрифов** за период → важные инсайты в §3.
- **Запись**: папка на дейлик (`YYYY-MM-DD/standup.md` + `sources/`), локальный git, шеринг —
  отдельный экспорт (`share.md`), не сам репо.
- **Латентность (быстрый/медленный слой + schedule)**: при on-demand сначала отдать быстрый ответ из
  готового — буфер (hook-дописывания во время работы) + последние `sources/` — за секунды; живой
  multi-agent `work-evidence` (минуты, время задаёт code-host-агент) — фоном, дополняет. Основной фикс —
  **schedule** (пре-счёт за 20–30 мин до синка), чтобы на встрече было мгновенно.
- **Роллап**: дейлики — feedstock для `retro` и `contribution-log` за большой период.
