# Daily workflow

Тонкий потребитель `work-evidence`: короткий период → 4-секционный стендап. Сбор и нормализацию
делает движок (`../work-evidence/references/workflow.md`); здесь — каденс, формат, запись.

## Канонические пути

`{data-repo}` — root приватного HARE Trail data repo (resolve через `HARETRAIL_DATA_DIR`,
затем текущий workspace, затем явный путь).

- Контейнер: `{data-repo}/daily/` (в корне data-repo).
- Дейлик = папка `{data-repo}/daily/YYYY-MM-DD/` → `standup.md` + `sources/`.
- Общие файлы контейнера: `README.md`, `tracker.md`, `journal.md`, опц. `_proposed-skill/`.

## Конфиг и первый запуск (`~/.haretrail/`)

- Каденс дейли (напр. `mon,thu`), источники, язык, фильтр — из конфига.
- **При первом запуске** спросить каденс/периоды и сохранить в конфиг.
- Окно по умолчанию: «с прошлого дейли по сегодня» = от даты предыдущего дейлика по каденсу.
  Если последний дейлик не записан в системе — снять окно по каденсу и **спросить, не взять ли раньше**.

## Латентность: быстрый/медленный слой + schedule

Боль: несхедуленный дейли на старте встречи = ждать минуты (живой multi-agent прогон, время задаёт
code-host-агент). Поэтому:

- **Быстрый слой (секунды):** собрать из готового — буфер (hook-дописывания во время работы) +
  последние `sources/` + последний дейлик; отдать сразу.
- **Медленный слой (минуты, фоном):** живой `work-evidence` — освежить/добрать пробелы; дополняет быстрый.
- **Schedule — основной фикс:** пре-счёт дейли по расписанию (за 20–30 мин до синка) → на встрече мгновенно.

## 1. Резолв периода

- Из каденса/конфига или явного запроса → конкретные `YYYY-MM-DD` (START..END).
- Зафиксировать в шапке: `Период покрытия: START – END`, `Записано: <дата>`.

## 2. Сбор evidence (через work-evidence)

- Вызвать движок `work-evidence` с пределами `{период: короткий, фокус: что-сделано}`.
- Сырьё по источникам движок складывает в `<дейлик>/sources/`.
- Дедуп и фильтр «личное/работа» — на стороне движка.

## 3. Рендер в 4 секции

1. **Прогресс по задачам** — по тикету; статус (done / in-progress / blocked), PR, коммит,
   путь по статусам + кто двигал (кто нажал кнопку ≠ чьё решение).
2. **Активности вне задач (invisible work)** — помощь коллегам (КАЖДАЯ отдельным пунктом: кто
   спросил → что я подсказал/сделал), координация, митинги; ревью чужих PR. Отдельным треком — то,
   что станет visible (воркшопы/доклады).
3. **Инсайты / проблемы / вопросы** — находки, риски, открытые вопросы к людям, уроки из
   `LESSONS.md`/дебрифов за период.
4. **Блокеры** — что мешает, от кого ждём.

Правила рендера:
- активность, относящаяся к тикету (даже если «выглядит» invisible) → в §1 под тикет;
- тон нейтральный, без сленга (отчёт могут показать лиду); «помог» только при подтверждённом
  результате; явная атрибуция КТО ЧТО;
- честность статусов: draft/open ≠ влито; статус — из системы, не из нарратива.

## 4. Записать в контейнер

- Создать `daily/YYYY-MM-DD/standup.md` (шапка + 4 секции) + `sources/` (сырьё движка).
- Обновить `tracker.md` (запись + статус источников) и `journal.md` (грабли, токены прогона).
- **Только операционное:** в `tracker.md`/`journal.md` контейнера пиши лишь про сами дейли (прогон, что
  собрали, грабли прогона, токены, статус источников). Разработку/дизайн самого скилла сюда НЕ писать →
  это в `{data-repo}/work-artifacts/2026-03-31-notes-system-foundation/` (tracker + `skills-design/`).
- `daily/` — локальный git-репо (`git init`, identity пользователя): закоммитить запись для rollback.
- **Шеринг**: при необходимости сделать отдельный экспорт `share.md` (язык по конфигу/Q6) —
  обезличенную выжимку для лида; не отдавать сам репо/`sources/`.

## 5. Роллап в contribution-log

Готовые `daily/*/standup.md` — feedstock для `contribution-log` за большой период: агрегировать их,
а не пересобирать с нуля (движок добирает только недостающее).

## 6. Что НЕ делать

- Не собирать «по памяти» в обход `work-evidence`.
- Не тащить личные проекты в рабочий дейли.
- Не приписывать себе чужие транзишены/решения; не выдавать draft/open за влитое.
- Не коммитить `sources/`-сырьё и приватику в шаринговый экспорт.
