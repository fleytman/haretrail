# Retro workflow

Рефлексивная ретроспектива за период. Объективный слой — движок `work-evidence`
(`../work-evidence/references/workflow.md`); здесь — дейлик-роллап, ЛС-майнинг, список чатов,
интерактивная проверка статуса и рендер.

## Канонические пути

`{data-repo}` — root приватного HARE Trail data repo (resolve через `HARETRAIL_DATA_DIR` и т.д.).

- Контейнер: `{data-repo}/retro/`. Ретро = папка `{data-repo}/retro/YYYY-MM-DD/` → `retro.md` + `sources/`.
- Дейлики (первичный источник): `{data-repo}/daily/*/standup.md`.

## Конфиг (`~/.haretrail/`)

- `HARETRAIL_RETRO_CADENCE` — каденс (дефолт: 2 недели + on-demand под демо/1:1).
- `HARETRAIL_RETRO_CHATS` — список чатов для скана проблем (собрать на первом запуске).
- Чат/трекер/code-host/язык — общие с `daily`/`work-evidence`.

## 0. Латентность: дейлики-first + два слоя + schedule

- **Приоритет — дейлики.** Ретро в первую очередь **роллапит готовые дейлики**
  (`{data-repo}/daily/*/standup.md` + их `sources/`) — чтение локального markdown, **секунды**, без
  живых агентов. Если дейлики собраны правильно, отдельный живой Slack/source-прогон НЕ нужен.
- **Два слоя:** быстрый (секунды) — из готовых дейликов; медленный (минуты, фоном) — живой
  `work-evidence` только чтобы освежить/добрать пробелы.
- **Schedule — основной фикс:** пре-счёт ретро по расписанию (до синка) → мгновенно.
- **Живой Slack-прогон = опциональное обогащение (safety-net).** «Боли команды» (широкий скан канала) —
  единственное, чего нет в дейликах из коробки: либо лёгкий скан вынести в дейли (по конфиг-чатам),
  либо оставить опц. фоновым в ретро.

## 1. Резолв периода

- Из каденса/конфига или запроса (часто «2 недели» / «спринт» / «к демо X»). → конкретные START..END.
- Зафиксировать в шапке `retro.md`: период покрытия, дата сбора, источники, что НЕ покрыто.

## 2. Источники

### 2.1 Дейлики (роллап + проверка разрешения)
- Собрать из `{data-repo}/daily/*/standup.md` в окне: §1 прогресс, §3 инсайты, §4 блокеры.
- Для каждого инсайта/блокера определить **статус**: решилось / ещё болит / неясно. Кросс-проверять
  поздними дейликами, трекером, code-host (через `work-evidence`).
- **Неясное — спросить у пользователя** («X решилось? Y ещё болит?»), не угадывать (можно пачкой в конце).

### 2.2 ЛС (личные переписки)
Майнить рабочие личные диалоги за период:
- жалобы (мне/на меня), поднятые мной проблемы/блокеры, заданные вопросы;
- открытые вопросы — мои без ответа + заданные мне без ответа;
- помощь мне → кого поблагодарить / что должен (reciprocity) — питает `contribution-log`.

### 2.3 Список чатов (из конфига)
- Пройтись по `HARETRAIL_RETRO_CHATS`: какие проблемы поднимали, **особенно в тредах с моим участием**.
- Опц. отдельный агент на **общий** поиск проблем в этих чатах (не только мои треды) — «боли команды».

### 2.4 Объективный слой
- `work-evidence` за период — что реально сделано/влито/закрыто (well + resolved).

## 3. Рендер (секции)

1. **Что прошло хорошо** — вины, влитое, доведённое.
2. **Что расстроило / болит** — провалы, факапы, что ещё ноет.
3. **Решилось vs ещё открыто** — статус инсайтов/блокеров из дейликов.
4. **Открытые вопросы** — мои (жду ответа) и мне (не закрыто).
5. **Кого поблагодарить / reciprocity** — помощь мне; что должен.
6. **Боли команды** — из списка чатов (контекст).
7. **Action items / что улучшить.**

Тон — нейтральный (демо/лид/1:1); атрибуция КТО ЧТО; «помог» только при подтверждённом результате.

## 4. Записать

- `{data-repo}/retro/YYYY-MM-DD/retro.md` (шапка + секции) + `sources/` (сырьё по источникам).
- Обновить `tracker.md`, `journal.md`. `retro/` — локальный git (rollback).
- **Только операционное:** в `tracker.md`/`journal.md` контейнера пиши лишь про сами ретро (прогон,
  что собрали, статус). Разработку/дизайн самого скилла сюда НЕ писать → это в
  `{data-repo}/work-artifacts/2026-03-31-notes-system-foundation/` (tracker + `skills-design/`).
- Что не успели/не покрыли — честно в §«не покрыли».

## 5. Связи

- **work-evidence** — общий движок сбора (объективный слой).
- **daily** — первичный источник (роллап); ретро ≠ дейли (рефлексия vs «что сделано»).
- **contribution-log** — ретро питает его (§5 благодарности/помощь, invisible work).

## 6. Что НЕ делать

- Не выдавать статус разрешения за факт без проверки/вопроса.
- Не майнить ЛС вне согласованного скоупа; не тащить личное в рабочий ретро.
- Не дублировать `contribution-log` целиком — отдавать ему благодарности/invisible как feedstock.
