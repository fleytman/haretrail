# Work Evidence

Движок сбора evidence о собственной работе за период. НЕ рендерит готовый отчёт — выдаёт
нормализованный, привязанный к источникам ledger, который потребляют тонкие скиллы
(`daily`, `contribution-log`).

Перед работой:

- Прочитай `../_shared/system-behavior.md` как общий reusable behavior contract.
- Прочитай `references/workflow.md`.
- Источники (трекер, чат, code-host), фильтр «личное/работа» и язык бери из конфига
  `~/.haretrail/` (см. workflow). Не хардкодить — спрашивать у пользователя и сохранять.

Используй этот движок, когда нужно:

- собрать «что сделано с X по Y» из объективных источников, а не «по памяти»;
- дать `daily` или `contribution-log` нормализованный материал;
- агрегировать готовые дейлики вверх в сводку за большой период (роллап).

Ключевые принципы:

- **Evidence-first**: code-host, трекер, чат, рабочие заметки/дебрифы — не догадки.
  По агенту на источник; сырой вывод каждого сохранять в `sources/`.
- **Тип участия и время действия**: различать created / merged-closed / committed (в т.ч.
  чужие) / reviewed / commented / pending; ключевать по ВРЕМЕНИ самого действия, а не по дате
  создания объекта.
- **КТО ЧТО**: фиксировать, кто действовал и кто кому помогал. **Кто нажал кнопку ≠ чьё
  решение**: чужой транзишен может быть и решением того человека, и фиксацией моего статуса.
- **Статус — из системы-источника, не из нарратива**; дедуп между источниками (один пункт
  всплывает в нескольких).
- **Только работа**: личное фильтровать по конфигу/эвристике.
- **«помог» только при подтверждённом результате**, иначе «помогал»/«подсказал».
